MarkItDown
MCP

这个实例运行的是 Microsoft MarkItDown 的 HTTP MCP 服务。根路径提供说明页，真正的 MCP 入口位于 /mcp 与 /sse。

Streamable HTTP

适合新的 MCP 客户端，直接把服务地址指向该端点。

/mcp

SSE

兼容基于 Server-Sent Events 的 MCP 调试与集成方式。

/sse

Workdir

懒猫数据目录被挂载到容器内 /workdir，可用文件 URI 访问。

file:///workdir/<filename> 
{
  "tool": "convert_to_markdown",
  "examples": [
    "https://example.com/report.pdf",
    "file:///workdir/sample.docx",
    "data:text/plain;base64,SGVsbG8sIE1hcmtJdERvd24h"
  ]
}
Inspector 调试 使用 MCP Inspector 时，把服务地址设置为当前应用域名下的 /mcp。
本地文件 先把待转换文件放到懒猫的 /lzcapp/var/data，再通过 file:///workdir/... 调用。
注意事项 该服务默认不带认证；如需公网使用，请结合上层路由或网关策略控制访问。
